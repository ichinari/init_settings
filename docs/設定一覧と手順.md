# 設定一覧と手順

現状の設定と、確認・追加で行う手順をまとめています。

---

## 1. Zsh

### 設定概要

- **.zshrc**: PATH、nvm、Powerlevel10k、**ccu**（codex/claude ログ解析）
- **.zprofile**: 必要に応じて

### 追加で行うこと

- 新規ターミナルを開くか、`source ~/.zshrc` で設定を反映

---

## 2. Neovim（NvChad ベース）

### 基本

- **Leader**: `<Space>`
- **init.lua**: Lazy で NvChad + custom プラグイン、Mason で LSP 導入

### 主要キーマップ（抜粋）

| キー | 動作 |
|------|------|
| **Space+e** | ファイルツリー（NvimTree）トグル |
| **Space+E** | 現在ファイルをツリーで表示 |
| **Space+t** | LazyGit（フロート） |
| **Space+ff** | Telescope ファイル検索 |
| **Space+fg** | Telescope ライブグレップ |
| **Space+fb** | バッファ一覧 |
| **Space+tt** | ToggleTerm |
| **Space+xx** | Trouble 診断一覧 |
| **Space+fm** | フォーマット（LSP/Conform） |
| **Space+fd** | 診断フロート |
| **gd** | 定義へジャンプ |
| **gr** | 参照検索（Telescope） |
| **K** | ホバー |
| **Space+rn** | リネーム |
| **Space+ca** | コードアクション |
| **[d / ]d** | 前/次の診断 |
| **Alt+h/j/k/l** | ウィンドウ移動（ノーマル・ターミナル） |

### LSP 設定（vim.lsp.config）

- **vtsls**: TypeScript/JavaScript（Mason の `vtsls` を絶対パスで実行、root は package-lock.json 等があるディレクトリ）
- **html**, **cssls**, **tailwindcss**: 有効化（cssls は Mason で CSS 用 LSP をインストールすること）

診断表示は Neovim 0.11 形式（下線・virtual text・サイン）で設定済み。

### TS（TypeScript）の確認手順

1. **front/ または backend/ 以下の .ts / .tsx を開く**  
   例: `nvim front/src/api/books/bookActions.ts`（プロジェクト root に `package.json` / `tsconfig.json` があること）
2. **`:LspInfo`** で **Active Clients** に **vtsls** が出ているか確認
3. 型エラーを意図的に入れて、下線・行末メッセージ・**Space+fd** のフロートで表示されるか確認

**vtsls が出ない場合**

- **`:Mason`** で **vtsls** がインストール済みか確認し、未ならインストール
- **`:LspLog`** で起動エラー（例: command not found）を確認
- Neovim を再起動してから再度ファイルを開く

### CSS の確認手順

1. **.css ファイルを開く**（例: `front/src/styles/globals.css`）
2. **`:LspInfo`** で **Active Clients** に **cssls** が出ているか確認
3. プロパティの補完・**Space+fm** で Prettier 整形が動くか確認

**cssls が出ない場合**

- **`:Mason`** で CSS 用 LSP（例: **css_lsp**）をインストール

### ファイルツリー

- **:NvimTreeToggle** または **Space+e**

## 3. 新規 PC セットアップ時の追加手順

1. **install.sh 実行後**
   - `source ~/.zshrc`
   - `git config --global user.email "..."` など
   - `nvm install --lts`
2. **Neovim 初回起動**
   - Lazy でプラグインが入るまで待つ
   - **`:Mason`** で **vtsls**（必須）、**css_lsp**（CSS を使う場合）、**html** / **tailwindcss** を必要に応じてインストール
## 4. トラブルシュート

| 現象 | 確認・対処 |
|------|------------|
| nvim で型エラーが表示されない | 該当ファイルが **package.json / tsconfig.json があるディレクトリ以下**か確認。**:LspInfo** で vtsls が Active か確認。**:Mason** で vtsls インストール。**:LspLog** でエラー確認 |
| vtsls が Active にならない | **:Mason** で vtsls をインストール。Neovim 再起動。`~/.local/share/nvim/mason/bin/vtsls` が存在するか確認 |
| CSS の LSP が効かない | **:Mason** で **css_lsp**（または vscode-css-language-server）をインストール。**:LspInfo** で cssls が Enabled/Active か確認 |
